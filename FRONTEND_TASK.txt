================================================================================

ОБЩЕЕ ОПИСАНИЕ
==============
Chippin — мобильное приложение (iOS + Android) для разделения счетов между
друзьями. Пользователь создаёт счёт, добавляет участников, позиции (вручную
или сканируя чек), делит позиции между участниками, указывает кто заплатил —
и получает итоговую сводку "кто кому сколько должен" с возможностью скачать PDF.

Бэкенд полностью готов (Laravel REST API). Ниже — все эндпоинты с точными
форматами запросов и ответов.

СТЕК ФРОНТЕНДА
==============
- Framework: Flutter (Dart)
- State Management: Riverpod или BLoC
- HTTP Client: Dio
- Auth: google_sign_in, sign_in_with_apple, flutter_telegram_login
- Камера: image_picker
- PDF: open_file или printing
- Шаринг: share_plus
- Хранилище: flutter_secure_storage (токен), shared_preferences (настройки)
- Навигация: GoRouter

================================================================================
  БЭКЕНД API — ПОЛНАЯ ДОКУМЕНТАЦИЯ
================================================================================

Базовый URL: http://10.0.2.2:8080/api  (для Android эмулятора)
             http://localhost:8080/api   (для iOS симулятора)

Все защищённые маршруты требуют заголовок:
  Authorization: Bearer {sanctum_token}
  Content-Type: application/json
  Accept: application/json

Формат успешного ответа:
  { "data": { ... }, "message": "..." }

Формат ошибки:
  { "error": "error_code", "message": "Описание ошибки" }

HTTP коды: 200 (ОК), 201 (создано), 401 (не авторизован), 403 (нет доступа),
           422 (ошибка валидации)

    "data": {
      "user": {
        "id": 1,
        "name": "Denis",
        "email": "denis@gmail.com",
        "avatar": "https://...",
        "provider": "google",
        "default_currency": "KZT",
        "language": "ru",
        "created_at": "2026-02-25T18:17:28.000000Z"
      },
      "token": "1|abc123..."    // <-- СОХРАНИТЬ В SECURE STORAGE
    },
    "message": "Authenticated successfully"
  }
  Ошибка 401: { "error": "auth_failed", "message": "Authentication failed: ..." }

GET /api/user
  Получить текущего пользователя
  Ответ 200:
  {
    "data": {
      "id": 1,
      "name": "Denis",

PUT /api/user
  Обновить профиль (все поля опциональны)
  Body: { "name": "Новое имя", "avatar": "https://..." }

POST /api/auth/logout

  Ответ 200: { "message": "Logged out successfully" }

POST /api/bills/{bill_id}/participants
  Добавить участника

  {

  Ответ 200: { "message": "Participant removed successfully" }

ПОЗИЦИИ (BILL ITEMS)
────────────────────────────────────────────────────────────────────────────────

POST /api/bills/{bill_id}/items
  Добавить позицию
  Body: { "name": "Пицца", "quantity": 2, "price_per_unit": 3500 }
  total считается автоматически (quantity × price_per_unit)
  Общий total счёта пересчитывается автоматически
  Ответ 201:
  {
    "data": { "id": 1, "name": "Пицца", "quantity": 2, "price_per_unit": "3500.00", "total": "7000.00", "splits": [] },
    "message": "Item added successfully"
  }

PUT /api/bills/{bill_id}/items/{item_id}
  Обновить позицию (все поля опциональны)
  Body: { "name": "Пицца Маргарита", "quantity": 1, "price_per_unit": 4000 }
  Ответ 200: { "data": { ...item }, "message": "Item updated successfully" }

DELETE /api/bills/{bill_id}/items/{item_id}
  Удалить позицию (total счёта пересчитывается)
  Ответ 200: { "message": "Item deleted successfully" }

────────────────────────────────────────────────────────────────────────────────
РАЗДЕЛЕНИЕ ПОЗИЦИЙ (SPLITTING)
────────────────────────────────────────────────────────────────────────────────

POST /api/bills/{bill_id}/items/{item_id}/split
  Разделить позицию между участниками

  Вариант 1 — Равное разделение (между ВСЕМИ участниками счёта):
  Body: { "type": "equal" }
  Бэкенд сам делит поровну между всеми участниками, последний получает остаток
  при нечётном делении (корректное округление)

  Вариант 2 — Ручное разделение:
  Body:
  {
    "type": "custom",
    "splits": [
      { "participant_id": 10, "quantity": 1 },
      { "participant_id": 11, "quantity": 1 }
    ]
  }
  ВАЛИДАЦИЯ: сумма quantity во всех splits должна равняться quantity позиции
  Каждый участник должен принадлежать этому счёту

  Ответ 200:
  {
    "data": {
      "item_id": 1,
      "splits": [
        { "id": 1, "participant_id": 10, "participant_name": "Denis", "quantity": 1, "amount": "3500.00" },
        { "id": 2, "participant_id": 11, "participant_name": "Аслан", "quantity": 1, "amount": "3500.00" }
      ]
    },
    "message": "Item split successfully"
  }

Цель: создать Flutter-проект с правильной структурой, настроить навигацию,
тему, Dio-клиент и базовые модели.

Шаги:
1. Создать Flutter-проект: flutter create chippin
2. Установить зависимости:
   - dio (HTTP клиент)
   - flutter_riverpod или flutter_bloc (state management)
   - go_router (навигация)
   - flutter_secure_storage (хранение токена)
   - shared_preferences (настройки)
   - google_sign_in
   - sign_in_with_apple
   - share_plus
   - image_picker
   - intl (форматирование дат и чисел)
3. Создать структуру папок (lib/config, models, services, providers, screens,
   widgets, utils)
4. Настроить тему (theme.dart):
   - Основной цвет: синий (#2563EB)
   - Минималистичный дизайн, чистые шрифты
   - Общие стили для кнопок, карточек, инпутов
5. Настроить GoRouter с маршрутами:
   /login, /home, /bills/new, /bills/:id/participants, /bills/:id/items,
   /bills/:id/split, /bills/:id/paid-by, /bills/:id/summary,
   /history, /profile
6. Создать Dio-клиент (api_client.dart):
   - baseUrl из конфига
   - Interceptor: добавляет Authorization header из secure_storage
   - Interceptor: при 401 — очищает токен, перенаправляет на /login
   - Interceptor: обработка ошибок сети
7. Создать модели данных (из JSON):
   - User (id, name, email, avatar, provider, default_currency, language)
   - Bill (id, name, date, currency, total, paid_by_participant_id,
           participants, items)
   - Participant (id, name, is_owner)
   - BillItem (id, name, quantity, price_per_unit, total, splits)
   - ItemSplit (id, participant_id, participant_name, quantity, amount)
   Каждая модель должна иметь fromJson/toJson

Проверка: приложение запускается, показывает пустой экран, Dio подключается
к http://10.0.2.2:8080/api/ping и получает "pong".

╔══════════════════════════════════════════════════════════════════════════════╗
║  ФАЗА 2: Авторизация (Login Screen)                                       ║
╚══════════════════════════════════════════════════════════════════════════════╝

Цель: экран логина с OAuth, сохранение токена, автоматический вход.

Экран: Login (/login)
  - Логотип "Chippin" по центру сверху
  - Подпись: "Split bills easily"
  - Три кнопки: "Sign in with Google", "Sign in with Apple",
    "Sign in with Telegram"
  - Каждая кнопка с иконкой провайдера слева
  - Нет полей email/пароля — только OAuth

Логика:
1. Пользователь нажимает кнопку → запускается OAuth через соответствующий пакет
2. Получаем access_token от провайдера
3. Отправляем на POST /api/auth/{provider} с { "token": "..." }
4. Получаем Sanctum токен в ответе → сохраняем в flutter_secure_storage
5. Сохраняем данные пользователя в провайдер
6. Навигация на /home

Автоматический вход:
- При запуске приложения проверяем наличие токена в secure_storage
- Если есть → вызываем GET /api/user
- Если 200 → навигация на /home
- Если 401 → удаляем токен, показываем /login

Проверка: вход через Google работает, токен сохраняется, повторный запуск —
автоматический вход.

╔══════════════════════════════════════════════════════════════════════════════╗
║  ФАЗА 5: Добавление участников                                            ║
╚══════════════════════════════════════════════════════════════════════════════╝

Цель: экран управления участниками.

Экран: Add Participants (/bills/{id}/participants)
  - Стрелка назад + заголовок "Add Participants"
  - Первый участник: "You" (владелец, создан автоматически) — без кнопки удаления
  - Поле ввода имени + кнопка "+" для добавления
  - Список участников, каждый с кнопкой удаления (X) кроме "You"
  - Кнопка "Next" — активна только когда >= 2 участников

Логика:
  - Данные участников берём из GET /api/bills/{id} (уже загружен при создании)

  - Каждое действие — мгновенный вызов API
  - Кнопка "Next" → /bills/{id}/items

  ВАЖНО: владелец (is_owner: true) отображается как "You" и НЕ удаляемый.
  Бэкенд вернёт 403 при попытке удалить владельца.

Проверка: добавление/удаление участников, кнопка Next активна при >= 2.

╚══════════════════════════════════════════════════════════════════════════════╝

Цель: экран добавления позиций с двумя способами ввода.

Экран: Add Items (/bills/{id}/items)
  - Стрелка назад + заголовок "Add Items"
  - Две кнопки вверху:
    * "Scan Receipt" (иконка камеры)
    * "Add Manually" (иконка карандаша)
  - Список добавленных позиций (имя, кол-во, цена, итого)
  - Каждая позиция — кнопки редактирования и удаления
  - Внизу: общая сумма + кнопка "Next"
  - "Next" активна при >= 1 позиции

Ручное добавление:
  - Показать поля: "Item name", "Quantity" (число), "Price per unit" (число)
  - Кнопка "Add" → POST /api/bills/{id}/items { name, quantity, price_per_unit }
  - Позиция появляется в списке

Сканирование чека:
  1. Открыть камеру через image_picker
  2. Сделать фото чека
  3. Показать спиннер "Scanning receipt..."
  4. Отправить: POST /api/bills/{id}/scan-receipt (multipart/form-data, поле image)
  5. Получить распознанные позиции
  6. Показать их в РЕДАКТИРУЕМОМ списке — пользователь может:
     - Изменить название, количество, цену
     - Удалить ненужные позиции
  7. Кнопка "Confirm" → POST /api/bills/{id}/items/bulk { items: [...] }
  8. Позиции добавляются к существующим

Редактирование позиции:
  - Нажатие на иконку редактирования → открыть диалог с полями
  - Сохранить: PUT /api/bills/{id}/items/{item_id}

Удаление позиции:
  - Нажатие на иконку удаления → подтверждение → DELETE /api/bills/{id}/items/{item_id}

"Next" → /bills/{id}/split

Проверка: ручное добавление работает, общая сумма обновляется.
(Сканирование чека можно протестировать если есть Claude API key в бэкенде)

╔══════════════════════════════════════════════════════════════════════════════╗
║  ФАЗА 7: Разделение позиций                                               ║
╚══════════════════════════════════════════════════════════════════════════════╝

Цель: экран распределения каждой позиции между участниками.

Экран: Split Items (/bills/{id}/split)
  - Стрелка назад + заголовок "Split Items"
  - Список всех позиций как карточки:
    * Название, количество, сумма
    * Статус: серый (не разделено) / зелёная галочка (разделено)
    * Если разделено — показать мелким шрифтом имена участников
  - Кнопка "Next" — активна ТОЛЬКО когда ВСЕ позиции разделены

Нажатие на позицию → Bottom Sheet с двумя опциями:

  Опция 1: "Split equally"
    - Одно нажатие → POST /api/bills/{id}/items/{item_id}/split { "type": "equal" }
    - Делится поровну между ВСЕМИ участниками
    - Bottom sheet закрывается, позиция помечается зелёной галочкой

  Опция 2: "Choose people" (ручное разделение)
    - Показать список всех участников

    Если quantity позиции > 1:
      - У каждого участника — числовое поле (+ / -) для ввода количества
      - Внизу счётчик: "Distributed: X of Y"
      - Кнопка "Done" активна только когда X == Y
      - Отправляем: POST /api/bills/{id}/items/{item_id}/split
        { "type": "custom", "splits": [{ "participant_id": 10, "quantity": 1 }, ...] }

    Если quantity позиции == 1:
      - У каждого участника — чекбокс
      - Выбранные делят стоимость поровну между собой
      - Бэкенд это обрабатывает: каждый выбранный получает quantity = 1,
        а amount = total / количество_выбранных
      - Отправляем так же с quantity: 1 для каждого выбранного

  Можно перераспределить: повторное нажатие на позицию заменяет старые сплиты.

"Next" → /bills/{id}/paid-by

Проверка: все позиции можно разделить, статусы обновляются, кнопка Next
активна только когда всё разделено.

Цель: итоговый экран со сводкой счёта.

Экран: Bill Summary (/bills/{id}/summary)

  - Секция "Items": список позиций с распределением (кто что взял)

    4. Пользователь выбирает WhatsApp, Telegram и т.д.
  - Кнопка "Done" → возврат на /home

API: GET /api/bills/{id}/summary (для долгов и долей)

Цель: экран профиля пользователя.

      просто URL), отправить PUT /api/user { "avatar": "https://..." }

  Нижняя навигация: Home, History, Profile (активно)
